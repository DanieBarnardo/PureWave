# The Tuning Questionnaire

Use this checklist before and during calibration. The aim is to understand what the system is doing now, where it is falling short, and what needs to be fixed before measurements start.

| Question / data point | Notes | Your notes / measurements |
| --- | --- | --- |
| Client name, site address, contact number, and preferred WhatsApp number | Basic session record. | |
| Which room and system are we tuning today? | Useful where there are multiple zones. | |
| What is frustrating you most about the system right now? | Let the client describe the pain in plain language first. | |
| If today goes well, what should sound or feel better tonight? | Outcome-first alignment. | |
| What do you use the room for most: movies, series, music, gaming, or mixed use? | Guides priorities during tuning. | |
| Which seat matters most, and are there other seats that also need to work well? | Defines the target listening area. | |
| Please list every source in regular use | Streaming box, TV apps, Blu-ray player, console, media PC, turntable, and so on. | |
| Please list the core signal chain | Display, AVR or processor, amps, speakers, subwoofers, sources, control system. | |
| Record make and model of AVR, processor, amps, speakers, subwoofers, and display | Necessary for capability checks. | |
| Has any auto-setup already been run? | Audyssey, Dirac, ARC, YPAO, MCACC, manual PEQ, or none. | |
| Are current settings saved anywhere? | Backup screenshots or profiles before changing anything. | |
| Are there any known faults or intermittent issues? | Buzzing drivers, loose terminals, HDMI dropouts, noisy fans, failing remotes. Fix these first. | |
| Are all speakers and subwoofers working reliably at normal volume? | Quick sanity check before deeper work. | |
| Where are the current subwoofer positions? | Measure and document before moving anything. | |
| Does each subwoofer have level, phase, crossover, polarity, EQ, or app control? | Needed for integration work. | |
| Are there room treatments already in place? | Note panels, curtains, carpets, bass traps, and furniture. | |
| Have any speakers been moved recently or mounted off-axis? | Mechanical placement matters as much as DSP. | |
| Are speaker distances, levels, and crossovers currently manual or automatic? | Understand the baseline. | |
| Are tone controls, dynamic EQ, loudness modes, night modes, or dialogue enhancers active? | These can disguise the real issue. | |
| Which sources sound best, and which sound worst? | Sometimes the problem is source-specific rather than room-wide. | |
| Are there lip-sync issues or switching delays? | Flag control and HDMI issues while you are there. | |
| Are projector alignment, focus, and aspect ratio also part of today’s brief? | Confirm whether video checks are expected. | |
| Can we access the AVR app, web UI, passwords, and advanced menus? | Do not begin tuning without access. | |
| Is there a reliable network connection if the platform needs it? | Especially relevant for Dirac and app-based control. | |
| Measure room dimensions and note major room features | Even on tuning jobs, room context matters. | |
| Note floor, wall, ceiling finishes, windows, and major reflective surfaces | Helps interpret the data later. | |
| Measure ambient noise floor with the room as normally used | Include HVAC state in your notes. | |
| Log starting master volume and reference behaviour | Useful if the client says the system never feels effortless. | |
| Capture pre-tuning measurements and screenshots | Frequency response, timing, sub integration, and any obvious nulls or ringing. | |
| After listening, what does the client notice first: too much bass, too little bass, poor dialogue, harshness, weak surround, or inconsistent seat-to-seat sound? | Prioritise the wins that matter to them. | |
| Are there household constraints on playback level? | Neighbours, children, sleeping spaces, and time-of-day limits matter. | |
| Does the client want one best preset or separate presets for movies, music, and late-night listening? | Good final alignment question before finalising changes. | |
